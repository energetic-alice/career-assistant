# LinkedIn Pack — MVP

Новый артефакт в текущем admin-боте: LinkedIn-аудит по методике Алисы + 5 вариантов headline под конкретного клиента.

## Scope MVP

1. **Аудит профиля** по чек-листу (30 баллов, 4 блока). Для каждого пункта — галочка (выполнено / нет) + конкретная рекомендация что сделать.
2. **5 вариантов headline** по формуле `[Грейд] [Должность] | [Стек] | [Индустрии] | [Опыт/достижение] | [Формат]`. Лимит 120 символов. С ключевыми словами под рекрутерский поиск.

Итоговый артефакт — Google Doc (+ сообщение в админ-чате с ссылкой и кнопкой «Отметить как отправлен клиенту»).

## Вход

Нужно хотя бы одно из:
- LinkedIn URL (`clientSummary.linkedinUrl`) — через существующий `linkedin-fetcher.ts` (cookie → Perplexity fallback)
- Резюме (`stageOutputs.resumeVersions`)

Плюс `clientSummary` (грейд, опыт, target-markets, english, `selectedTargetRoles`) для контекста.

Если нет ни LinkedIn, ни резюме — кнопка отдаёт ошибку «нужна или ссылка на LinkedIn, или резюме».

## Чек-лист аудита (30 баллов)

Источник — методика Алисы. Промпт обязан использовать РОВНО этот чек-лист, без отсебятины.

### Блок 1 — Базовая информация (10 баллов)

- [ ] Профессиональное фото (не селфи, не на фоне обоев или леса) — **2 балла**
- [ ] Фоновая картинка (баннер) заполнена — **1 балл**
- [ ] Заголовок содержит специальность + ключевые навыки — **2 балла**
- [ ] Город и страна указаны корректно — **1 балл**
- [ ] Контактная информация доступна — **1 балл**
- [ ] Ссылки на портфолио / GitHub / сайт добавлены — **2 балла**
- [ ] URL профиля кастомизирован (не набор цифр) — **1 балл**

### Блок 2 — Раздел «О себе» (8 баллов)

- [ ] Раздел заполнен (не пустой) — **2 балла**
- [ ] Есть конкретные навыки и технологии — **2 балла**
- [ ] Указаны достижения с цифрами — **2 балла**
- [ ] Есть призыв к действию (как с вами связаться) — **2 балла**

### Блок 3 — Опыт работы (7 баллов)

- [ ] Заполнен опыт работы за последние 5+ лет — **2 балла**
- [ ] Описание проектов и достижения, а не просто обязанности — **2 балла**
- [ ] Используются ключевые слова из вакансий — **2 балла**
- [ ] Нет пробелов в карьере без объяснений — **1 балл**

### Блок 4 — Навыки и рекомендации (5 баллов)

- [ ] Добавлено 10+ навыков — **1 балл**
- [ ] Топ-3 навыка подтверждены коллегами — **2 балла**
- [ ] Есть хотя бы 2 рекомендации от коллег / руководителей — **2 балла**

**Максимум: 30 баллов.**

## Формула headline

```
[Грейд (если есть)] [Должность (самое популярное название)] | [Стек или 2-4 ключевые технологии] | [Индустрии] | [Число лет опыта ИЛИ самое крутое достижение] | [Формат: например B2B Remote — при отсутствии разрешения на работу в EU при желаемом зарубежном рынке]
```

Требования:
- **≤ 120 символов**. Если Claude вернул больше — регенерим до 2 раз.
- Ключевые слова для поиска рекрутерами (используй популярные формулировки из вакансий target-роли).
- 5 вариантов с разным акцентом, чтобы клиент мог выбрать:
  1. классика: грейд + должность + стек
  2. с крутым достижением вместо стажа
  3. с акцентом на индустрию
  4. B2B / Remote (если target-рынок ≠ страны, где клиент может работать)
  5. keyword-heavy — под ATS и фильтры рекрутеров

## Архитектура

```
Карточка клиента ──кнопка──> runLinkedinPack ──> fetchInputs (LI + резюме + summary)
                                             ──> phase1_audit (чеклист + рекомендации)
                                             ──> phase2_headline (5 вариантов, валидация 120 симв)
                                             ──> renderer → markdown
                                             ──> createGoogleDoc
                                             ──> сохранение в stageOutputs.linkedinPack
                                             ──> сообщение куратору со ссылкой
```

### Pipeline stages

- `linkedin_generating` — запущен
- `linkedin_ready` — Google Doc создан, ждёт отправки
- `linkedin_sent` — куратор отправил клиенту (необратимо, как `final_sent`)
- `linkedin_failed` — упало

## Roadmap (вне MVP)

- **v2 — тексты секций**: copy-paste для About (200-400 слов), Experience bullets, Top-50 Skills.
- **v3 — контент-план**: 4-8 недель, темы по target-role + экспертиза + «человеческие» посты.
- **v4 — генерация постов**: draft + 2-3 варианта по выбранному пункту плана.
- **v5 — self-service**: отдельный client-facing entry point, если зайдёт у клиентов.

## Ограничения и риски

- LinkedIn cookie протухает ~1-2 мес — fallback на Perplexity уже есть; если и там пусто, аудит делаем по резюме + помечаем «LinkedIn спарсить не удалось, проверь руками».
- Headline > 120 симв — в коде ловим Zod-refine'ом и регенерим (максимум 2 раза).
- Часть пунктов чеклиста (фото, баннер, кастом URL) не проверяется автоматически по тексту — модель должна честно пометить «не проверяется — проверь руками».
